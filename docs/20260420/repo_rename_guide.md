# GitHub 저장소 및 로컬 폴더 이름 변경 절차 안내

본 문서는 GitHub에 있는 저장소(repository)의 이름을 변경하고, 로컬 PC 상의 해당 폴더 이름도 함께 변경하는 절차를 단계별로 안내합니다. Git의 정상적인 동작을 유지하면서 안전하게 이름을 수정할 수 있습니다.

---

### 1. GitHub 저장소 이름 변경

1. GitHub 웹사이트에 로그인합니다.
2. 이름을 변경할 저장소로 이동합니다.
3. 상단 메뉴에서 `Settings` 탭을 클릭합니다.
4. `Repository name` 항목에서 현재 이름을 새 이름으로 수정합니다.
5. `Rename` 버튼을 클릭하여 변경을 완료합니다.
6. 변경 후, 저장소는 새 이름으로 즉시 접근 가능해집니다.

> 참고: 이름 변경 후 기존의 URL은 자동으로 새 URL로 리디렉션되지만, 원격 연결을 사용하는 모든 클라이언트는 새 URL로 업데이트해야 정상 작동합니다.

---

### 2. 로컬 저장소의 원격 저장소 주소 업데이트

GitHub 저장소 이름이 변경되면, 로컬 저장소가 가리키는 원격 저장소(remote)의 URL도 새 이름에 맞게 업데이트해야 합니다.

1. 로컬 PC에서 터미널 또는 명령 프롬프트를 실행합니다.
2. 해당 프로젝트 폴더로 이동합니다.
   ```bash
   cd /경로/로컬-저장소-폴더
   ```
3. 현재 원격 저장소 주소를 확인합니다.
   ```bash
   git remote -v
   ```
4. 원격 저장소 주소를 새 저장소 이름으로 변경합니다.
   ```bash
   git remote set-url origin https://github.com/사용자이름/새-저장소-이름.git
   ```
5. 변경이 정상적으로 반영되었는지 다시 확인합니다.
   ```bash
   git remote -v
   ```

이후 `git push`, `git pull` 등 모든 원격 연동 명령이 새 저장소 이름으로 정상 작동합니다.

---

### 3. 로컬 폴더 이름 변경

로컬 파일 시스템 상에서 저장소 폴더의 이름을 변경하는 것은 안전하며, Git 동작에 영향을 주지 않습니다.

#### 변경 방법

- **파일 탐색기 사용 시**
  - 폴더를 마우스 오른쪽 클릭 후 ‘이름 바꾸기’를 선택하고 새 이름을 입력합니다.

- **터미널 사용 시 (리눅스, macOS)**
  ```bash
  mv 기존-폴더-이름 새-폴더-이름
  cd 새-폴더-이름
  ```

> 주의: `.git` 폴더가 폴더 내부에 그대로 유지되도록 주의해야 하며, 삭제 또는 이동하지 않아야 합니다.

---

### 4. 변경 후 동작 확인

1. 간단한 파일 수정 후 커밋을 수행합니다.
   ```bash
   git add .
   git commit -m "이름 변경 후 테스트 커밋"
   ```
2. 정상적으로 푸시되는지 확인합니다.
   ```bash
   git push origin main
   ```
   (브랜치 이름이 `main`이 아닌 경우 해당 브랜치 이름으로 변경)

---

### 5. 참고 사항

- 로컬 폴더 이름 변경은 Git 동작과 무관하므로 자유롭게 수행 가능합니다.
- 원격 저장소 이름 변경 후, 원격 URL을 업데이트하지 않으면 `push` 또는 `pull` 시 오류가 발생합니다.
- 팀 프로젝트의 경우, 모든 팀원이 원격 저장소 URL을 업데이트해야 정상적인 협업이 가능합니다.
- IDE에서 프로젝트를 열고 있다면, 폴더 이름이 변경된 경우 새 경로로 프로젝트를 다시 열어야 할 수 있습니다.

---

이상의 절차를 따르면 GitHub 저장소 이름과 로컬 폴더 이름 모두 안전하게 변경할 수 있습니다.
